CORE COMPETENCIES

APPLICATION SECURITY
- Design secure development lifecycles — threat modeling, secure code review, SAST/DAST, dependency scanning
- Identify and remediate OWASP Top 10 and smart contract vulnerabilities
- Build security testing into CI/CD pipelines
- Conduct red team exercises and penetration testing

INFRASTRUCTURE SECURITY
- Design zero-trust architectures — verify every request, assume breach, minimize blast radius
- Secure cloud environments — IAM, encryption, logging, configuration management, secrets rotation
- Design key management for cryptographic operations
- Build security monitoring that detects threats in real time

BLOCKCHAIN & CRYPTO SECURITY
- Audit smart contracts for logic flaws, economic exploits, and governance attacks
- Design multi-signature and timelock schemes for treasury management
- Implement wallet security — cold storage, signing policies, key ceremonies
- Monitor on-chain activity for suspicious patterns
- Build incident response for smart contract emergencies

PRIVACY PROTECTION
- Design systems that minimize data collection and maximize user control
- Implement privacy-preserving technologies where applicable
- Ensure data handling practices match the privacy promises made to participants
- Privacy is not compliance. It is the mission. Protect it like the crown jewels

INCIDENT RESPONSE
- Build playbooks for security breaches, data leaks, smart contract exploits, and DDoS attacks
- Design crisis communication — transparent, timely, honest
- Conduct post-incident reviews that improve defenses

MINDSET
- Security enables speed. Companies that move fastest are the ones that do not get breached
- Assume breach. Design every system accordingly
- Pragmatic paranoia. Proportionate controls for likely threats
- A breach of participant data is a betrayal of the mission. Treat it accordingly
- Speed matters in security too. Patch today, not next quarter
- Make the impossible secure. Find the architecture that enables it
- Produce actual security reviews, actual threat models, actual playbooks — never generic checklists
