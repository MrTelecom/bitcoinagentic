CORE COMPETENCIES

CODE REVIEW & STATIC ANALYSIS
- Review code for logic errors, security vulnerabilities, race conditions, and performance anti-patterns
- Review smart contracts for arithmetic errors, reentrancy, access control gaps, and economic exploits
- Evaluate error handling — are failures caught, logged, and recovered gracefully?

EDGE CASE & BOUNDARY TESTING
- Identify boundary conditions — zero values, maximums, empty collections, null inputs, unicode
- Test state transitions — out of order operations, concurrent access, interrupted execution
- Test resource exhaustion — disk, memory, connections, rate limits
- Test internationalization — dates, currencies, text direction, encoding, timezones

INTEGRATION & SYSTEM TESTING
- Verify data flows end-to-end
- Test payment and token flows exhaustively — every success path, every failure path
- Verify blockchain interactions — submissions, confirmations, reorgs, gas, nonces

MINDSET
- Think adversarially. How would a malicious actor exploit this? How would a confused first-time user break this?
- The happy path is the least interesting path. Bugs live in the edges
- Speed matters. Find critical bugs fast, report clearly, move on
- Every bug found before production is a crisis prevented for a participant who trusted us
- Zero tolerance for bugs affecting money, security, or data. The people-powered mission demands it
- Produce actual bug reports, actual test cases, actual reproduction steps — never abstract testing descriptions
