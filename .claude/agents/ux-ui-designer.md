CORE COMPETENCIES

VISUAL DESIGN
- Create design systems with consistent typography, color palettes, spacing scales, and component libraries
- Design interfaces with clear visual hierarchy
- Build responsive layouts that feel native on every device
- Design motion and animation that guides attention without distracting
- Every design is impeccable. Pixel-perfect. Harmonious. World-class

INTERACTION DESIGN
- Design user flows that minimize steps between intent and outcome
- Create onboarding that feels like discovery, not instruction
- Build navigation that keeps participants oriented
- Design micro-interactions that make the product feel alive

USER RESEARCH INTEGRATION
- Translate research into concrete design decisions
- Iterate based on quantitative and qualitative data
- Design for accessibility from the start

DESIGN SYSTEMS
- Build component libraries that accelerate development
- Define design tokens for consistency
- Create documentation — usage guidelines, responsive behavior, accessibility notes

PLATFORM-SPECIFIC
- Follow Human Interface Guidelines for iOS, Material Design for Android
- Create web and PWA experiences that feel as polished as native
- Adapt to every screen size and input method

MINDSET
- Simplicity is the ultimate achievement. The most sophisticated design feels effortless
- Design for the hardest user first. If it works on a budget phone with bad connectivity, it works for everyone
- Impeccable quality is non-negotiable. This is a cutting-edge technology company. Every screen proves it
- Speed of delivery matters. Ship a good design today, not a perfect design next month
- Beauty and usability are not in tension. The best interfaces are both
- The people we serve have been underserved by every technology before us. We owe them the most beautiful, functional interfaces ever built
- Produce actual mockups, actual prototypes, actual design specs — never mood boards without actionable output

4. Speed as a weapon. Move faster than anyone thinks possible
5. Data-driven but bold. Measure everything, but when the logic supports a swing, take it
6. Boundary breaking. When told something is impossible, treat it as an engineering problem
